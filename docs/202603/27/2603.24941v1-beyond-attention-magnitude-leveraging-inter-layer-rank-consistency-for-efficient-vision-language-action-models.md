## 摘要
视觉-语言-动作（VLA）模型在机器人操控方面表现出色，但由于处理密集的视觉标记（tokens），面临显著的推理延迟。现有的标记削减方法主要依赖注意力幅度作为静态选择标准。在这项工作中，我们对这一假设提出了挑战，揭示了高注意力标记具有任务依赖性，甚至可能降低策略性能。为解决这一问题，我们引入了 TIES（Tau 引导的层间高效选择），这是一个由层间标记排名一致性引导的动态框架。通过自适应地平衡注意力幅度与排名一致性，TIES 在无需额外训练的情况下确保了稳健的标记选择。在 CogACT + SIMPLER 基准测试中，TIES 在减少 78% 标记使用的同时，将平均成功率提高了 6%，并在多种解码器和基准测试中展现出强大的泛化能力。

## Abstract
Vision-Language-Action (VLA) models excel in robotic manipulation but suffer from significant inference latency due to processing dense visual tokens. Existing token reduction methods predominantly rely on attention magnitude as a static selection. In this work, we challenge this assumption, revealing that high-attention tokens are task-dependent and can even degrade policy performance. To address this, we introduce \textbf{TIES} (\textbf{T}au-guided \textbf{I}nter-layer \textbf{E}fficient \textbf{S}election), a dynamic framework guided by inter-layer token ranking consistency. By adaptively balancing attention magnitude with ranking consistency, TIES ensures robust token selection without requiring additional training. On the CogACT + SIMPLER benchmark, TIES improves average success rates by 6\% while reducing token usage by 78\%, and demonstrate strong generalization across diverse decoders and benchmarks.

*   **核心问题**：现有的视觉标记缩减方法大多假设“注意力幅度（Attention Magnitude）越大，标记越重要”。作者挑战了这一假设，发现高注意力标记往往具有任务依赖性，在某些情况下甚至会关注无关的背景或伪特征（Spurious Features），从而误导策略并降低成功率。
*   **研究动机**：寻找一种比单纯的注意力强度更可靠、更鲁棒的指标，以实现高效且不损失性能（甚至提升性能）的视觉标记剪枝。

### 2. 论文提出的方法论：TIES 框架
*   **核心思想**：引入**层间排名一致性（Inter-layer Rank Consistency）**作为衡量注意力可靠性的指标。
*   **关键技术细节**：
    *   **Kendall 秩相关系数 ($\tau$)**：用于量化相邻 Transformer 层之间高注意力标记排名的稳定性。
    *   **伪相关锁定假设（Spurious Locking Hypothesis）**：作者发现，当 $\tau$ 值较高时，模型往往陷入局部最优，固定关注冗余或有害特征；而较低的 $\tau$ 值代表模型在迭代更新关注点，属于健康的“信息流”。
    *   **动态选择策略**：
        *   **Hard-TIES**：基于阈值的二元选择。若 $\tau$ 过高（不可信），则放弃注意力排名，改用均匀采样（Uniform Sampling）以保证多样性；否则保留高注意力标记。
        *   **Soft-TIES**：通过线性插值动态分配高注意力标记与多样性标记（如均匀采样或 DivPrune）的比例。
    *   **相似性触发机制**：利用视频流的临时冗余，仅在当前帧与锚点帧视觉相似度低于阈值时才重新计算 $\tau$，进一步降低计算开销。
*   **算法流程**：离线校准（确定 $\tau$ 的分布） $\rightarrow$ 在线推理（计算当前帧 $\tau$） $\rightarrow$ 动态调整保留标记的来源 $\rightarrow$ 执行前向传播。
