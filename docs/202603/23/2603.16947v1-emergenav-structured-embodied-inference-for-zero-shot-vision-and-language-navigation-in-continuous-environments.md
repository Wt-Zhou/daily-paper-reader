## 摘要
连续环境下的零样本视觉语言导航（VLN-CE）对于现代视觉语言模型（VLM）而言仍然具有挑战性。尽管这些模型编码了有用的语义先验，但其开放式推理并不能直接转化为稳定的长程具身执行。我们认为，关键瓶颈不仅在于知识的缺失，更在于缺乏一种用于组织指令遵循、感知落地、时间进度和阶段验证的执行结构。我们提出了 EmergeNav，这是一个将连续 VLN 形式化为结构化具身推理的零样本框架。EmergeNav 结合了用于阶段结构化执行的“计划-解决-转换”（Plan--Solve--Transition）层级结构、用于目标条件感知提取的 GIPE、用于进度落地的对比双记忆推理，以及用于时间对齐局部控制和边界验证的角色分离双视野（Dual-FOV）传感。在 VLN-CE 任务上，EmergeNav 仅使用开源 VLM 骨干网络，在无需特定任务训练、显式地图、图搜索或航路点预测器的情况下，实现了强大的零样本性能，在 Qwen3-VL-8B 上达到 30.00 SR，在 Qwen3-VL-32B 上达到 37.00 SR。这些结果表明，显式的执行结构是将 VLM 先验转化为稳定具身导航行为的关键要素。

## Abstract
Zero-shot vision-and-language navigation in continuous environments (VLN-CE) remains challenging for modern vision-language models (VLMs). Although these models encode useful semantic priors, their open-ended reasoning does not directly translate into stable long-horizon embodied execution. We argue that the key bottleneck is not missing knowledge alone, but missing an execution structure for organizing instruction following, perceptual grounding, temporal progress, and stage verification. We propose EmergeNav, a zero-shot framework that formulates continuous VLN as structured embodied inference. EmergeNav combines a Plan--Solve--Transition hierarchy for stage-structured execution, GIPE for goal-conditioned perceptual extraction, contrastive dual-memory reasoning for progress grounding, and role-separated Dual-FOV sensing for time-aligned local control and boundary verification. On VLN-CE, EmergeNav achieves strong zero-shot performance using only open-source VLM backbones and no task-specific training, explicit maps, graph search, or waypoint predictors, reaching 30.00 SR with Qwen3-VL-8B and 37.00 SR with Qwen3-VL-32B. These results suggest that explicit execution structure is a key ingredient for turning VLM priors into stable embodied navigation behavior.

### 2. 方法论：核心思想与关键技术
EmergeNav 的核心是一个**Plan–Solve–Transition (PST)** 层级结构，配合感知与记忆机制：
*   **PST 层级结构**：
    *   **Plan（计划）**：将长指令分解为一系列基于锚点（Anchor）的子目标序列。
    *   **Solve（解决）**：在当前子目标下进行局部控制，通过 ReAct 风格的循环（推理-动作-观察）生成动作。
    *   **Transition（转换）**：独立的验证模块，利用全景感知判断当前阶段是否完成，决定是否切换到下一个子目标。
*   **GIPE（目标条件感知提取）**：一种提示词层级的协议，强制 VLM 仅提取与当前子目标相关的视觉证据，避免冗余信息干扰决策。
*   **对比双记忆推理（Contrastive Dual-Memory）**：
    *   **STM（短时记忆）**：记录子目标内的密集动作轨迹和视觉变化。
    *   **LTM（长时记忆）**：存储已验证完成的阶段性视觉快照。通过对比 STM 和 LTM，智能体能判断自己是在前进、停滞还是回退。
*   **角色分离的双视野（Dual-FOV）传感**：
    *   **局部控制（Solve）**：使用前向 30° 三视图（低冗余，高频）。
    *   **边界验证（Transition）**：使用 360° 全景视图（全局信息，低频）。

### 6. 主要结论与发现
*   **结构化执行是关键**：显式的执行结构（PST）能显著提升开源 VLM 在复杂导航任务中的稳定性。
*   **性能突破**：EmergeNav 在不使用显式地图、图搜索或航路点预测器的情况下，使用 Qwen3-VL-32B 达到了 **37.00 SR**，显著优于之前的零样本方法。
*   **感知与记忆的互补性**：GIPE 主要提升感知的相关性（决策质量），而双记忆机制主要提升长程稳定性和执行效率（减少无效探索）。

### 7. 优点（亮点）
*   **无需外部辅助工具**：不依赖显式地图构建、拓扑图或预训练的航路点预测器，保持了纯 VLM 策略的简洁性。
*   **感知效率高**：通过角色分离的 Dual-FOV 设计，平衡了局部控制的实时性和全局验证的准确性。
*   **可解释性强**：PST 架构使得导航过程可审计，能够清晰看到智能体在哪个子目标阶段出错。

### 8. 不足与局限
*   **路径效率（SPL）仍有提升空间**：虽然成功率（SR）很高，但 SPL 略低于部分基准，说明在子目标切换时可能存在短暂的绕路或灵敏度不足。
*   **骨干网络依赖**：性能高度依赖于 VLM 的基础能力，在较小规模的模型上，子目标边界的判定精度仍存在挑战。
*   **计算延迟**：虽然感知经过了优化，但频繁调用大型 VLM 进行推理在实时机器人部署中可能仍面临延迟压力。
