## 摘要
近期配备流匹配 (Flow Matching, FM) 动作头的视觉-语言-动作 (VLA) 模型在复杂机器人操作中取得了最先进的性能。然而，FM 所需的多步迭代 ODE 求解引入了推理延迟，阻碍了响应式物理控制。虽然目前的加速工作主要优化视觉-语言模型 (VLM) 主干网络，但动作头的瓶颈仍被忽视。为解决这一问题，我们提出了 ProbeFlow，这是一种专为连续机器人控制量身定制的无须训练的自适应推理框架。通过利用初始速度向量与前瞻速度向量之间的余弦相似度来评估几何轨迹复杂度，ProbeFlow 动态调度积分步数以剪枝冗余的网络评估。在 MetaWorld 基准测试中，它将动作解码速度提高了 14.8 倍（平均步数从 N = 50 减少到 2.6），并在不降低操作成功率的情况下将端到端系统延迟降低了 2.8 倍。在长时程 LIBERO 基准测试中，探针会自动分配更密集的调度以应对语义瓶颈，从而有效解决流求解器的延迟问题。真实世界的物理部署证实，ProbeFlow 在确保执行稳定性的同时，成功缓解了动作解码延迟，为低延迟连续生成策略提供了一种高度实用的解决方案。

## Abstract
Recent Vision-Language-Action (VLA) models equipped with Flow Matching (FM) action heads achieve state-of-the-art performance in complex robot manipulation. However, the multi-step iterative ODE solving required by FM introduces inference latency that precludes responsive physical control. While current acceleration efforts optimize the Vision-Language Model (VLM) backbone, the action head bottleneck remains overlooked. To address this, we propose ProbeFlow, a training-free adaptive inference framework tai- lored for continuous robotic control. By evaluating geometric trajectory complexity via the cosine similarity between initial and lookahead velocity vectors, ProbeFlow dynamically sched- ules integration steps to prune redundant network evaluations. On the MetaWorld benchmark, it accelerates action decoding by 14.8x (reducing average steps from N = 50 to 2.6) and cuts end-to-end system latency by 2.8x without compromising the manipulation success rate. On the long-horizon LIBERO benchmark, the probe automatically allocates a denser schedule to navigate semantic bottlenecks, effectively resolving the flow solver delay. Real-world physical deployments confirm that ProbeFlow successfully mitigates action decoding latency while ensuring execution stability, offering a highly practical solution for low-latency continuous generative policies.

*   **研究动机**：当前的视觉-语言-动作（VLA）模型（如 OpenVLA）常采用流匹配（Flow Matching, FM）作为动作头，以生成高质量的连续控制轨迹。然而，FM 依赖于多步迭代的常微分方程（ODE）求解（通常需要 20-50 步），这导致了严重的推理延迟，无法满足机器人实时物理控制的严苛要求。
*   **核心问题**：现有的加速研究多集中在优化 VLM 主干网络，而忽略了动作头迭代解码这一瓶颈。论文旨在寻找一种无需重新训练、能根据轨迹复杂度动态调整计算量的加速方案。

#### 2. 方法论
*   **核心思想**：利用 FM 轨迹的几何特性——在简单的过渡运动中路径趋于直线，而在复杂的交互（如抓取）中路径弯曲。通过评估局部轨迹的“线性度”来动态分配积分步数。
*   **关键技术细节**：
    *   **前瞻线性探针 (Lookahead Linearity Probe)**：在起始点 $t=0$ 计算初始速度 $v_{start}$，然后尝试向前迈出一大步（如 $\Delta t=0.5$）到达探测点，计算该点的速度 $v_{probe}$。
    *   **余弦相似度评估**：计算 $v_{start}$ 与 $v_{probe}$ 之间的余弦相似度 $S$。若 $S \approx 1$，说明轨迹非常直，可以跳过中间步骤；若 $S \ll 1$，说明轨迹弯曲，需要加密采样。
    *   **自适应步数调度**：根据 $S$ 将步数 $N$ 映射在 $[N_{min}, N_{max}]$ 之间。
    *   **状态复用机制**：在计算探针时产生的速度向量会被直接用于最终的动作合成，确保探针本身的计算开销被完全吸收，不会引入额外负担。
*   **算法流程**：单次推理中，先执行一次前瞻探测，根据相似度决定是采用“极简两步法”直接输出结果，还是启动“密集积分模式”以保证精度。

*   **公平性**：对比了同等计算预算下的固定步数方法（如 $N=3$），证明了 ProbeFlow 的成功率提升源于自适应调度而非模型本身的鲁棒性。实验设计客观，涵盖了仿真与实物。

#### 6. 主要结论与发现

*   **即插即用**：无需对现有 VLA 模型进行任何微调或重新训练，部署成本极低。
*   **几何直观**：利用余弦相似度作为曲率的代用指标，计算极其轻量且物理意义明确。
*   **计算效率极高**：通过状态复用，使得“探测”过程几乎是免费的（Zero-overhead probing）。

#### 8. 不足与局限
*   **超参数敏感性**：线性阈值 $\epsilon$ 在不同领域（如 MetaWorld 到 LIBERO）之间需要手动微调以获得最佳平衡，未能实现完全自动化的超参数自适应。

*   **应用限制**：目前主要针对流匹配（Flow Matching）架构，对于传统的扩散模型（Diffusion Policy）可能需要调整几何评估逻辑。
